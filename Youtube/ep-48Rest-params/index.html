Rest Params

The rest parameter in JavaScript (...) allows a function to accept an indefinite number of arguments as an array. It is used to group the remaining arguments passed to a function into a single array, after all the defined parameters. This is particularly useful when the number of arguments is not known in advance or for handling variadic functions (functions that can take a variable number of arguments). The rest parameter must always be the last parameter in a function definition.

Difference b/w Rest params and spread operator

Rest Parameter
1. Gathers multiple elements into an array
2. Used in function parameters
3. Can only appear as the last parameter
4. Converts an indefinite number of arguments into an array

Spread Operator
1. Expands elements into individual elements
2. Used in function calls, arrays, and objects
3. Can appear anywhere in the list of arguments/elements
4. Spreads out elements of an array or object
